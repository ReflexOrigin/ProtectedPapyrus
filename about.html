AES Encryption Details

For each user note, a unique Encryption Key and Initialization Vector (IV) are generated. These are securely stored in the database. When creating or updating a note, the application uses the unique Encryption Key and IV associated with that note. The note's content is encrypted using the AES algorithm in Cipher Block Chaining (CBC) mode, and the encrypted content is stored in the database. When accessing a note, the application retrieves the unique Encryption Key and IV for that note from the database, decrypts the content using the AES algorithm in CBC mode, and presents the decrypted content to the user for viewing or editing.

The Encryption Keys are stored securely in the database, ensuring that only authorized users with the correct keys can decrypt and access the note content. AES is a widely recognized and secure encryption algorithm, and using unique Encryption Keys and IVs for each note enhances security, preventing one compromised key from affecting other notes.

Secure Connection Details

Protected Papyrus prioritizes the establishment of a secure and reliable connection to ensure the confidentiality and integrity of user data. The implementation of a robust secure connection is crucial for safeguarding sensitive information within the application. The following key measures are employed to fortify the connection:

  • Network Connectivity Assurance: Protected Papyrus operates with a stringent network connectivity policy, allowing access only through a secure internet connection.
  • Real-time Internet Check: The application performs periodic checks, verifying internet connectivity every 10 seconds.
  • Hash and Salt Algorithm for Login Security: User login credentials are fortified using a robust hash and salt algorithm.
  • User Data Privacy: Each user is assigned a unique identifier, ensuring a personalized and private user experience.
  • Scheduled Notes and Encryption: Users can schedule notes for future delivery to other users, enhancing communication flexibility. The application employs AES encryption for note content, ensuring end-to-end security during storage and transmission.

Secure connections in Protected Papyrus are foundational to our commitment to user privacy and data protection. By integrating advanced encryption, user authentication, and network security measures, we strive to create a trustworthy platform for secure note-taking.

Secured-Note Sharing Details

Secure Note Sharing in our application allows users to share notes with enhanced security and privacy. Each shared note undergoes encryption with a unique key, ensuring confidentiality. The process involves secure storage of encrypted notes and keys in our database, and recipients must be verified before gaining access.

Users can create secure notes within the application by selecting the "Create New Note" option. During the creation process, they can choose to enable secure sharing. Every note is encrypted with a unique key using advanced encryption algorithms, adding an extra layer of security. Encrypted notes and their corresponding keys are securely stored in our database, employing industry-standard security measures to safeguard user data.

To share a secure note, users navigate to the note they want to share and select the "Share" option. They then enter the recipient's information, such as their email address or username. Before granting access to the shared note, the application verifies the identity of the recipient. Once the recipient's identity is verified, they gain access to the shared note. Recipients can view the shared note content in a secure environment. If allowed by the sender, recipients may also have the option to edit the note, maintaining collaborative capabilities while prioritizing security and privacy.
